Inventory Admin CRUD Demo

หน้าเว็บตัวอย่างนี้ทำมาให้ตาม flow ของแอดมินคลังสินค้า: ดู supplier, ดูสินค้า, สร้างและแก้ supplier, สร้างและแก้ product, สร้าง restock order, เพิ่มสินค้าเข้า order, แล้วเปลี่ยนสถานะเป็น Received เพื่อให้ stock เพิ่มตาม logic ฝั่ง Django.

ถ้าเปิดไฟล์นี้จาก browser ตรง ๆ ให้ระวังเรื่อง CORS ด้วย และถ้า backend ยังไม่เปิด POST/PATCH สำหรับ supplier กับ product ปุ่ม create / edit จะยิงแล้วได้ 405 หรือ 404.

API snippets ที่ก็อปไปใช้ต่อได้

เอาไปย้ายต่อใน React / JS ฝั่งฟร้อนได้เลย
Create supplier 
async function createSupplier(payload) {
  const res = await fetch(`${BASE_URL}/api/stock/suppliers/`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(payload),
  });
  if (!res.ok) throw new Error(await res.text());
  return await res.json();
}
Update supplier 
async function updateSupplier(supplierId, payload) {
  const res = await fetch(`${BASE_URL}/api/stock/suppliers/${supplierId}/`, {
    method: "PATCH",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(payload),
  });
  if (!res.ok) throw new Error(await res.text());
  return await res.json();
}

Flow ที่หน้าเว็บนี้ใช้

ยึดตามสิ่งที่แอดมินคลังน่าจะทำจริง
Step 1: โหลด master data: products, categories, suppliers, warehouses, warehouse locations, employees, restocks
Step 2: สร้างหรือแก้ supplier ได้จาก form ข้างตาราง
Step 3: เตรียมหน้า warehouse และ warehouse location ให้กด edit / delete ได้ในไฟล์เดียว
Step 4: สร้างหรือแก้ product ได้จาก form ข้างตาราง inventory
Step 5: สร้าง restock order แล้วเพิ่มสินค้าเข้า order
Step 6: เปลี่ยน status เป็น Received เพื่อให้ stock เพิ่มอัตโนมัติ
ถ้า backend ของ branch ที่ใช้อยู่ยังเป็น read-only สำหรับ supplier หรือ product ให้แก้ฝั่ง Django ก่อน แล้วค่อยใช้หน้าเว็บนี้ต่อ.
