Inventory & Restock Admin Demo

หน้าเว็บนี้ออกแบบตามโฟลว์ที่ backend ปัจจุบันทำได้จริง: ดูสต๊อก, ดู location, สร้าง restock order, เพิ่มสินค้าเข้า order, และเปลี่ยนสถานะเป็น Received เพื่อให้ stock เพิ่มตาม logic ฝั่ง Django.

ถ้าจะเปิดไฟล์นี้แยกจาก Django ตรง ๆ ผ่าน browser ให้ระวังเรื่อง CORS ด้วย เพราะ settings ตอนนี้อนุญาต origin แค่ localhost:3000 และ 127.0.0.1:3000. ถ้าจะเทสง่ายสุด ให้รันฟร้อนบนพอร์ต 3000 หรือเอาไฟล์นี้ไปวางเป็น template ฝั่ง Django.

API snippets ที่ก็อปไปใช้ต่อได้

เขียนเป็น fetch แบบ vanilla JS เพื่อให้ย้ายไป React/Vue/Next ได้ง่าย
1) ดึงสินค้า 
const BASE_URL = "http://127.0.0.1:8000";

async function getProducts() {
  const res = await fetch(`${BASE_URL}/api/catalog/products/`);
  if (!res.ok) throw new Error("Load products failed");
  return await res.json();
}
2) ดึง supplier 
async function getSuppliers() {
  const res = await fetch(`${BASE_URL}/api/stock/suppliers/`);
  if (!res.ok) throw new Error("Load suppliers failed");
  return await res.json();
}
3) สร้าง restock order 
async function createRestock(payload) {
  const res = await fetch(`${BASE_URL}/api/stock/restocks/`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(payload),
  });
  if (!res.ok) throw new Error(await res.text());
  return await res.json();
}

// payload example
// {
//   supplier: 1,
//   employee: 1,
//   location: 1,
//   restock_date: "2026-04-10",
//   restock_status: "Pending"
// }
4) เพิ่มสินค้าเข้า restock 
async function addRestockDetail(payload) {
  const res = await fetch(`${BASE_URL}/api/stock/restock-details/`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(payload),
  });
  if (!res.ok) throw new Error(await res.text());
  return await res.json();
}

// payload example
// {
//   restock: 3,
//   product: 12,
//   quantity: 25
// }
5) เปลี่ยนสถานะเป็น Received 
async function markRestockReceived(restockId) {
  const res = await fetch(`${BASE_URL}/api/stock/restocks/${restockId}/`, {
    method: "PATCH",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ restock_status: "Received" }),
  });
  if (!res.ok) throw new Error(await res.text());
  return await res.json();
}

Flow ที่หน้าเว็บนี้ใช้

ยึดตามสิ่งที่ backend ปัจจุบันเปิดให้ใช้ได้จริง
Step 1: โหลด master data ได้แก่ products, categories, suppliers, employees, locations และ restocks
Step 2: แอดมินเลือก supplier / employee / location เพื่อสร้าง restock order
Step 3: แอดมินเพิ่มสินค้าแต่ละตัวด้วย endpoint restock-details
Step 4: เมื่อของเข้าจริง ให้เปลี่ยนสถานะเป็น Received
Step 5: ฝั่ง Django จะเพิ่ม stock_quantity ของสินค้าใน order ให้อัตโนมัติ
ตอนนี้หน้าเว็บนี้ยังไม่ได้ทำ CRUD สินค้าโดยตรง เพราะ ProductViewSet ใน repo ปัจจุบันเป็น ReadOnlyModelViewSet. ถ้าจะทำหน้า “จัดการสินค้าเต็มรูปแบบ” จริง ๆ ต้องเปลี่ยนฝั่ง backend ก่อน.
